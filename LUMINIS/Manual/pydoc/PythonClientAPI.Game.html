get_shortest_path(start, end, avoid)[source]¶

Returns the shortest path between start and end. If avoid is not None, the method will return the shortest path between start and end that does not visit any points in avoid.

If there is no path, None is returned.

Note: The path-finding algorithm may come at a cost of performance. For a quicker path look-up alternative, use get_next_point_shortest_path.

Parameters:
  • start ((int,int)) – source
  • end ((int,int)) – target
  • avoid (set) – a set of (x,y) tuples to exclude from path-finding
Returns:

list of points representing the shortest path such that the first element in the list is the next point in the path, or None.
Return type:

list of point

move(unit, position)[source]¶

Moves a unit one step closer to position. Use this method to assign moves for your units.

Note: this method uses get_next_point_in_shortest_path, which does not take into account dynamic game elements (i.e. other units or nests) that may stand in the way between the source and destination. It only takes into account walls and tiles.

To include a list of “avoid” points in path-finding, use get_shortest_path.

Parameters:
  • unit (FriendlyUnit) – unit to move
  • position ((int,int)) – target position
Returns:

whether or not the move will result in MOVE or REST
Return type:

MoveType
